## 目标
- 升级直接依赖中已过时的包并做必要代码兼容，减少安装与构建的警告与潜在风险。
- 顺带移除 `cnpm` 脚本以避免命中淘系镜像导致的证书错误。

## 升级列表
- `uuid`：^8 → ^9（需要代码改动，CommonJS 导入方式变化）
- `electron-builder`：^23 → ^24（解决 `electron-osx-sign` 的弃用链路）
- 脚本：将 `cnpm run ...` 改为 `npm run ...`

## 代码改动
- `electron/notebookFile.js`：
  - 将 `const uuid = require('uuid')` 改为 `const { v4: uuidv4 } = require('uuid')`
  - 把所有 `uuid.v4()` 替换为 `uuidv4()`

## 变更文件
- `package.json`：升级版本并调整脚本
- `electron/notebookFile.js`：兼容 v9 导入与调用

## 验证
- 删除 `node_modules` 与 `package-lock.json` 后执行 `npm install`
- 确认安装日志不再出现 `uuid@3`/`electron-osx-sign` 弃用警告（可能仍有第三方子依赖，但我们已清理直接依赖）
- 运行开发与打包脚本：`npm run start`、`npm run estart`、`npm run package-mac`（仅本机可用）

确认后将按以上修改直接落地。
