# UI Design Capability

## ADDED Requirements

### Requirement: 极简主义视觉风格
系统 SHALL 采用极简主义设计风格，专注于内容而非装饰。

#### Scenario: 中性配色方案
- **WHEN** 用户打开应用
- **THEN** 应使用中性色系（灰白黑）作为主色调
- **AND** 应使用单一强调色（绿色 #25b864）用于交互元素
- **AND** 不应出现紫色渐变或其他花哨配色

#### Scenario: 简洁的视觉元素
- **WHEN** 用户浏览界面
- **THEN** 应使用纯色背景（不应有渐变动画）
- **AND** 阴影应简化为单层或边框
- **AND** 圆角应统一为 4px-8px
- **AND** 不应有玻璃态效果（backdrop-filter）

### Requirement: 性能优化的样式
系统 SHALL 避免使用高性能开销的 CSS 效果。

#### Scenario: 流畅的滚动和交互
- **WHEN** 用户滚动内容或切换页面
- **THEN** 应感觉流畅无卡顿
- **AND** 不应使用 `backdrop-filter: blur()`（高 GPU 开销）
- **AND** 不应使用持续运行的背景动画
- **AND** 不应在悬停时使用 `transform: scale()`（触发重排）

#### Scenario: 简化的动画
- **WHEN** 界面出现动画
- **THEN** 应仅包含必要的过渡动画（Modal、页面切换）
- **AND** 动画时长应 ≤ 300ms
- **AND** 不应有装饰性动画（背景渐变、按钮缩放）

### Requirement: 合理的布局比例
系统 SHALL 优化界面布局，提升空间利用率和可读性。

#### Scenario: 侧边栏宽度
- **WHEN** 用户查看侧边栏
- **THEN** 宽度应为 64px（紧凑而不拥挤）
- **AND** 应使用纯色背景（不使用玻璃态）

#### Scenario: 编辑器宽度和留白
- **WHEN** 用户编辑文档
- **THEN** 内容区最大宽度应为 720px（最佳阅读宽度）
- **AND** 左右留白应至少 2rem
- **AND** 行高应为 1.6-1.8（提升可读性）

#### Scenario: 文件列表布局
- **WHEN** 用户浏览文件列表
- **THEN** 列表项应简洁（无过多阴影和圆角）
- **AND** 间距应统一使用 4px 倍数

### Requirement: 品牌一致性
系统 SHALL 保留品牌色 #25b864 作为唯一强调色。

#### Scenario: 强调色使用
- **WHEN** 用户看到交互元素（按钮、链接、选中状态）
- **THEN** 应使用绿色 #25b864
- **AND** 不应使用紫色或其他渐变色
- **AND** 强调色应用于重要操作（新建、保存、确认）

### Requirement: 暗黑模式支持
系统 SHALL 继续支持暗黑模式，并适配新配色方案。

#### Scenario: 暗黑模式配色
- **WHEN** 用户切换到暗黑模式
- **THEN** 应使用深色背景（#171717）和浅色文字（#fafafa）
- **AND** 强调色应保持 #25b864（或适当调亮）
- **AND** 边框和分隔线应使用深色中性色（#404040）

#### Scenario: 暗黑模式可读性
- **WHEN** 用户在暗黑模式下阅读
- **THEN** 对比度应符合 WCAG AA 标准（≥ 4.5:1）
- **AND** 代码块应使用深色主题配色
